Rescue Animal Management System

The Rescue Animal System was originally developed as a console-based application and later enhanced to improve usability, structure, and maintainability. The system manages rescue animal records and allows users to perform key operations such as adding, reserving, and viewing animals.

To improve user experience, the application was redesigned with a graphical user interface using Java Swing. This transition required restructuring the program to separate user interface logic from core functionality, resulting in a more modular and maintainable codebase. Additionally, input validation and error handling were implemented to ensure data accuracy and prevent invalid user input.

Enhancements

  • Converted console-based system to Java Swing GUI
  • Refactored code into modular and maintainable structure
  • Implemented input validation and error handling
  • Improved user interaction and usability

Skills Demonstrated

  • Software design and modular development
  • User interface development

Algorithms & Data Structures

Enhancements to this system focused on improving efficiency and performance through the implementation of more appropriate data structures. The original system relied on linear searches using ArrayLists, resulting in O(n) time complexity for retrieving and updating records.

To optimize performance, HashMaps were introduced to enable constant-time O(1) lookup operations. This significantly improved the speed of searching, updating, and removing records. ArrayLists were retained for iteration and display purposes, creating a balanced design that supports both efficient access and structured output.

Enhancements

  • Replaced linear search with HashMap for O(1) lookup
  • Improved performance of data retrieval and update operations
  • Implemented dual data structures (ArrayList + HashMap)

Skills Demonstrated

  • Time complexity analysis and optimization (Big-O)
  • Data structure selection and implementation
  • Performance optimization techniques

Databases

The system was enhanced to incorporate persistent data storage, allowing information to be retained and retrieved across application sessions. In the original implementation, all data was stored in memory, resulting in data loss when the application terminated.

To address this limitation, a file-based persistence layer was implemented to simulate database functionality. This enhancement introduced structured data storage and enabled the system to support full CRUD operations, including creating, reading, updating, and deleting records. Data integrity was further improved by ensuring that all modifications are consistently saved, maintaining synchronization between application state and stored data.

Enhancements

  • Implemented file-based database for persistence
  • Developed full CRUD operations
  • Ensured data integrity through consistent saving

Skills Demonstrated

  • Data persistence and storage design
  • CRUD operation implementation
  • Database design concepts
