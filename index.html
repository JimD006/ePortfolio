Software Design & Engineering

Rescue Animal Management System

The original system was a console-based application that relied on a menu-driven interface and tightly coupled logic. This structure limited usability and made the code difficult to maintain and extend.

To address these limitations, the system was redesigned using a graphical user interface built with Java Swing. This required separating the user interface from the underlying business logic, resulting in a more modular and maintainable design. Input validation was also introduced to ensure consistent and accurate data entry, improving overall system reliability.

These enhancements reflect a shift from a simple academic implementation to a more structured application that follows real-world software engineering practices, including modular design, separation of concerns, and improved user interaction.

Algorithms & Data Structures

The original implementation relied on ArrayLists and linear search operations, resulting in O(n) time complexity for locating and updating records. While functional, this approach becomes inefficient as the dataset grows.

To improve performance, HashMaps were introduced to provide constant-time O(1) lookup capabilities. This significantly enhanced the efficiency of operations such as searching, reserving, and removing animals. ArrayLists were retained to support iteration and structured display of records, allowing the system to balance performance with usability.

This enhancement demonstrates the importance of selecting appropriate data structures based on system requirements and highlights how performance can be improved through algorithm optimization.

Skills Demonstrated

  • Time complexity analysis and optimization (Big-O)
  • Data structure selection and implementation
  • Performance optimization techniques

Databases

In the original system, all data was stored in memory, which meant that information was lost once the application was closed. This limitation prevented the system from functioning as a persistent solution.

To address this, a file-based persistence layer was implemented to simulate database functionality. This allowed data to be saved and reloaded between application runs. Full CRUD operations were also introduced, enabling users to create, retrieve, update, and delete records while maintaining consistency between memory and stored data.

These improvements demonstrate core database concepts, including data persistence, storage design, and data integrity, and reflect how real-world systems manage and protect information.

Skills Demonstrated

  • Data persistence and storage design
  • CRUD operation implementation
  • Data integrity and synchronization
